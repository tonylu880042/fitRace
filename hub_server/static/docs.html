System Overview系統概觀

FitRaceStudio turns ordinary fitness equipment into live race stations. Data flows through the system like this:

Equipment → BLE / FTMS → Antenna Board → UART → Edge Node → Wi-Fi → Central Hub → Dashboards

  • Antenna Board — a dual-radio BLE receiver wired to the Edge Node. Each board channel (uart-1, uart-2) can hold up to 3 BLE connections.
  • Edge Node — a Raspberry Pi that collects equipment telemetry and forwards it to the Hub. It hosts its own device page on port 8001.
  • Central Hub — runs the race engine, dashboards, and this admin console on port 8000.

FitRaceStudio 把一般健身器材變成即時競賽站位。資料流向如下:

健身器材 → BLE / FTMS → 天線板 → UART → Edge Node → Wi-Fi → 中央 Hub → 各儀表板

  • 天線板 — 接在 Edge Node 上的雙射頻 BLE 接收板。每個通道(uart-1、uart-2)最多同時連 3 台 BLE 設備。
  • Edge Node — 樹莓派,負責收集器材遙測並轉送給 Hub;本身在 8001 埠提供裝置設定頁。
  • 中央 Hub — 執行競賽引擎、儀表板與本控制台(8000 埠)。

Initial Setup初次設定

  1. Power up the venue access point first. Every device (hub, edge nodes, tablets, dashboard screen) joins the same venue Wi-Fi.
  2. Power up the Central Hub, then the Edge Nodes. They join the venue Wi-Fi automatically once provisioned.
  3. On a tablet or laptop joined to the same Wi-Fi, open http://<hub-ip>:8000/static/systemAdmin.html.
  4. Check the status strip: Edge Nodes online count, Wi-Fi signal, update state. All healthy → you are ready to bind equipment.

Maintenance Unlock

Protected actions (station assignment, Wi-Fi changes, updates, power) require the maintenance access code. Click Maintenance Unlock in the top bar and enter the code provided with your system. The code is remembered by this browser until cleared.

Equipment Binding設備綁定

Binding tells an Edge Node which BLE equipment belongs to which antenna channel. It is done on the Edge device page — from Edge Nodes, click Open Device Page.

  1. Pick the antenna channel (uart-1 / uart-2) and press SCAN.
  2. The scan wizard lists nearby FTMS equipment. Pick a device, give it a name (e.g. TREAD_01), choose its type and channel, then confirm.
  3. Press Save bindings. The Edge runtime restarts automatically and the antenna boards reconnect to the saved targets.

Unbinding

Unbind removes one device; Clear all bindings removes every binding and disconnects all devices from the antenna boards. Both update the boards immediately and restart the runtime.

LimitEach antenna channel supports at most 3 connected devices. A fourth binding on the same channel will not connect — spread equipment across channels.
Good to knowIdle machines stop sending data but stay connected. "idle" in the equipment table means no recent data — it does not mean the link is lost.

綁定的目的,是告訴 Edge Node 哪台 BLE 器材屬於哪個天線通道。操作在 Edge 裝置頁進行 — 在 Edge Nodes 頁點 Open Device Page。

  1. 選擇天線通道(uart-1 / uart-2)後按 SCAN。
  2. 掃描精靈會列出附近的 FTMS 器材。選擇設備、命名(例如 TREAD_01)、選類型與通道,然後確認。
  3. 按 Save bindings 儲存。Edge runtime 會自動重啟,天線板隨即重新連上已儲存的目標。

解除綁定

解綁移除單一設備;清空全部綁定移除所有綁定並讓天線板斷開全部設備。兩者都會立即更新天線板並重啟 runtime。

限制每個天線通道最多連 3 台設備;同通道的第 4 個綁定不會連上 — 請把器材分散到不同通道。
小知識閒置中的器材會停止傳資料,但連線仍然存在。設備表格顯示「idle」代表最近沒有資料,不代表斷線。

Station Assignment站位指派

A station is a numbered race position shown on the dashboard. Assigning maps a telemetry stream (a bound machine) to a station number. Open Stations in the sidebar.

  1. Set the station number with the − / + stepper.
  2. Pick the telemetry stream (e.g. TREAD_01 (treadmill)).
  3. Press Assign. The station appears in the table below with a green Assigned dot.

Use Copy Signup Link to copy the athlete signup URL for the selected station — open it on the athlete's phone or the venue tablet. In the table, ✎ loads a row back into the form and ✕ unassigns the station.

站位是儀表板上帶編號的比賽位置。指派就是把某條遙測資料流(已綁定的器材)對應到站位編號。從側欄開啟 Stations。

  1. 用 − / + 調整站位編號。
  2. 選擇遙測資料流(例如 TREAD_01 (treadmill))。
  3. 按 Assign。站位會出現在下方表格,亮綠色 Assigned 燈。

Copy Signup Link 會複製該站位的選手報名網址 — 用選手手機或場地平板開啟即可報名。表格中 ✎ 會把該列帶回上方表單編輯,✕ 解除該站位指派。

Wi-Fi NetworkWi-Fi 網路

The Network page shows the hub's live Wi-Fi state: signal quality, RSSI, SSID, and IP. The same panel exists on each Edge device page for the edge's own connection.

  1. Press Choose Wi-Fi Network. A picker opens and scans (about 10 seconds).
  2. Pick a network. Secured networks you have not joined before ask for a password.
  3. Press Connect. The result, including the new IP, is shown in the picker.
WarningChanging Wi-Fi can change the hub's IP address. Browsers on other devices will lose the page — reconnect using the new IP shown after connecting. All race devices must share one network.

Network 頁顯示 Hub 的即時 Wi-Fi 狀態:訊號品質、RSSI、SSID 與 IP。每台 Edge 的裝置頁也有相同面板管理自己的連線。

  1. 按 Choose Wi-Fi Network,選單開啟並掃描(約 10 秒)。
  2. 選擇網路;沒連過的加密網路會要求密碼。
  3. 按 Connect。連線結果(含新 IP)會顯示在選單內。
注意切換 Wi-Fi 可能改變 Hub 的 IP,其他裝置上開著的頁面會斷線 — 請用連線後顯示的新 IP 重新開啟。所有比賽設備必須在同一個網路。

Software Updates軟體更新

The hub checks the release server once at startup and then weekly. The Software page shows the current version, the latest available versions, and when the last check ran.

  1. Check Now forces an immediate check (requires internet on the hub).
  2. When an update is available, the Download → Install Hub → Apply Hub buttons appear. Run them in order.
  3. Apply Hub restarts the hub service and is only allowed while the race state is IDLE.

All release artifacts are signature-verified before installation.

Hub 會在開機時檢查一次、之後每週自動檢查更新。Software 頁顯示目前版本、可用的最新版本與上次檢查時間。

  1. Check Now 立刻強制檢查(Hub 需可連外網)。
  2. 有可用更新時會出現 Download → Install Hub → Apply Hub 按鈕,依序執行。
  3. Apply Hub 會重啟 hub 服務,只有比賽狀態為 IDLE 時才允許。

所有更新檔在安裝前都會做簽章驗證。

Power Controls電源控制

Open with the red System Override button. All actions are blocked unless the race state is IDLE.

ActionEffect
Restart Hub ServiceRestarts the hub software only — fastest recovery for a stuck console

由紅色 System Override 按鈕開啟。比賽狀態必須是 IDLE 才能執行任何動作。

動作效果
Restart Hub Service只重啟 hub 軟體 — 控制台卡住時最快的恢復方式

Troubleshooting疑難排解

Edge node shows offline

  1. Check the edge Pi has power and its Wi-Fi LED activity.
  2. Open the edge device page (http://<edge-ip>:8001/) — if it loads, check its Wi-Fi panel signal.
  3. Weak signal (below −75 dBm): move the edge or the access point; avoid metal equipment between them.

Equipment shows "idle" or "no data"

  1. Make sure someone is actually using the machine — idle machines stop transmitting but remain connected.
  2. On the edge device page, run STATUS on the antenna channel: it lists which targets are truly connected.
  3. If the board answers slowly or devices will not reconnect, use REBOOT on that channel — the board recovers within seconds and auto-reconnects saved targets.

Wi-Fi scan shows very few networks

If only the current network appears, rescan once; the first scan after startup can be cached.

Reporting a problem

Open Support (bottom of the sidebar), press Copy System Report, and send the snapshot to the FitRace team with a short description of what you were doing.

Edge Node 顯示離線

  1. 確認 Edge 樹莓派有電、Wi-Fi 燈有在閃。
  2. 開啟 Edge 裝置頁(http://<edge-ip>:8001/)— 開得起來就檢查它的 Wi-Fi 訊號面板。
  3. 訊號太弱(低於 −75 dBm):移動 Edge 或分享器位置,避免中間隔著金屬器材。

設備顯示 idle 或 no data

  1. 先確認真的有人在用那台器材 — 閒置的器材會停止傳資料,但連線還在。
  2. 在 Edge 裝置頁對該天線通道執行 STATUS:會列出實際連線中的目標。
  3. 若天線板回應變慢或設備一直連不回來,對該通道下 REBOOT — 板子幾秒內恢復,並自動回連已儲存的目標。

Wi-Fi 掃描只出現很少網路

若只看到目前連線中的網路,請再按一次 Rescan;開機後的第一次掃描可能拿到快取結果。

回報問題

開啟側欄底部的 Support,按 Copy System Report,把系統快照連同操作情境一起傳給 FitRace 團隊。
